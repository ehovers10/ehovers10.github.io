Semantics of propositional logic

Defining properties of propositions

Properties of propositions

  • Just as in categorical logic, we can use our semantics for propositional logic to define properties of propositions.
    • Tautology: A proposition is tautologous, if it is impossible for it to be false.
    • Contradiction: A proposition is contradictory, if it is impossible for it to be true.
    • Equivalence: Two propositions are equivalent, if they are true in all the same situations and false in all the same situations.
    • (In)consistency: Two propositions are (in)consistent if it is (im)possible for both to be true simultaneously.

Properties and truth tables

Determine what property (if any) a particular WFF has using its truth table.

  • Tautology: A proposition is tautologous if its column in the truth table has a T in every row.
  • Contradiction: A proposition is contradictory if its column has an F in every row.
  • Equivalence: Two propositions are equivalent, if their columns have the same truth value in every row.
  • Consistency: Two propositions are consistent, if they both have a T in at least one row.
    • Inconsistency means that in no row do both have a T.

Assessing arguments

Truth tables for arguments

  • Arguments aren't any different from complex propositions, as far as PL is concerned.
  • So, we can create truth tables for arguments the same way we did for complex propositions
  • But we can condense them a little bit.
  • Leave out the columns for individual propositional variables.
  • Just create a column for every premise in the argument.
  • And finish with a column for the conclusion.
  • Add indicators in each column to tell what it role it plays in the argument.

Indirect truth tables

Speeding up the process

Indirect truth tables

  • Truth tables can be somewhat time consuming.
    • This is especially true when there are a lot of propositional variables in a complex WFF or a lot of premises in an argument.
  • Indirect truth tables provide us with a quicker way to determine properties such as:
    • Validity
    • Tautology
    • Contradiction
  • If we know what property we are looking for, we only need one line of the truth table to determine whether it holds.

Indirect truth tables

Validity

  • If an argument is valid, then you know that it's impossible for the premises to be true without the conclusion also being true.
  • So, to test this, we attempt to make all the premises true, but have the conclusion be false.
  • If we can do this consistently, the argument is invalid.
  • If we run into a contradiction, it is valid.

Indirect truth tables

Tautology

  • If a WFF is tautological, then it can't possibly be false.
  • So, to test this, we attempt to make it false.
  • If we can do this then it is not true in all situations.
  • If we run into a contradiction, then Bob's your uncle.

Indirect truth tables

Contradiction

  • If a WFF is contradictory, then it can't possibly be true.
  • So, to test this, we attempt to make it true.
  • If we can do this then it is actually consistent.
  • If we run into a contradiction, then there you have it.
